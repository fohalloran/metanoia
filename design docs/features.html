Metanoia

, from Greek metanoein meaning change one's mind

Context and scope

The aim of this project is to create a sprint retrospective tool that can help scrum teams to streamline their retrospective process. It will do this by allowing teams to have flexibility over their boards, ensuring that previous actions are reminded of and performed and that all team members have an equal opportunity to share feedback with the team

Design goals

  • Team should have rolling list of incomplete actions
  • There should be flexibility surround anonymisation and other team norm preferences
  • There should be a distinct difference between scrum leadership accounts and others
  • Once a retrospective is complete, they should have the ability to freeze the board

Technical goals

  • Cloud based - scalable, fault tolerant
  • Containerised services
  • API Analytics output
  • Real time editing of notes - team members can be notes as they're added

Design

Pages

  • Retro board select
  • Retro session
    • 4 Columns
      • Successes
      • improvements
      • New actions
      • Previous actions
    • Ability to freeze board at end of session
    • Beginning of session, each user should be given a code to access the board, to save time on navigating. When admins starts session, then link the session to a team to get previous actions.
  • Board specific settings
  • Global settings

APIs

There should be an API for analytics output. As each scrum team can't be compared to eachother, the analytics should be performed only internally for a particular scrum team. This may be because teams have differing normals in terms of granularity of improvements/actions/successes

Data Storage

A file based data storage system such as MongoDB many of the disadvantages to MongoDB are less impactful on this project, such as lack of joins. Using a file based storage method means that I can have a file for each board, which will make storage very simple
